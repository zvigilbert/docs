The Redwood Cloud Portal gives administrators visibility and control over every environment in their organization's Redwood subscription. From the **Environments** page you can review configuration details, monitor health, and access environment-level settings — without contacting Redwood Support for routine information.

## View environment details

Each environment in the portal has a detail view with the full set of configuration and status information for that instance.

<Steps>
  <Step title="Open the Environments page">
    In the left navigation, select **Environments**.
  </Step>
  <Step title="Select an environment">
    Click the environment name to open its detail view.
  </Step>
  <Step title="Review the details">
    The detail view displays the environment's name, product, type (Production or Non-Production), hosting region, current status, and any configuration notes.
  </Step>
</Steps>

The detail view is read-only for most fields — core configuration such as region and product version is managed by Redwood as part of your subscription. Settings you can influence are described in [Manage environment-level settings](#manage-environment-level-settings) below.

## Understand environment health and status

The **Status** field tells you the current operational state of an environment:

| Status | What it means |
|---|---|
| **Active** | The environment is running normally and ready to use. |
| **Maintenance** | Redwood is performing scheduled maintenance or an upgrade. The environment is temporarily unavailable. |
| **Degraded** | The environment is reachable but not performing normally. Redwood support is typically already investigating. |
| **Unavailable** | The environment cannot be reached due to an incident or infrastructure issue. |

<Tip>
  For production environments, check the status before coordinating critical business activities. If a production environment is in **Maintenance**, check your scheduled upgrade windows to understand when it will return to **Active**.
</Tip>

<Warning>
  A **Degraded** status does not always trigger an automatic notification. If you notice degraded performance that is not reflected in the portal status, report it to Redwood Support immediately so the team can investigate.
</Warning>

## Manage environment-level settings

Some settings are configurable at the environment level directly from the portal. What is available depends on your product and subscription:

- **Display name** — You can rename an environment to make it easier to identify in the list (for example, renaming "Environment A" to "RunMyJobs Production - US East").
- **Upgrade windows** — For each environment, you can configure the maintenance windows that control when Redwood applies product upgrades. See [Upgrades](/upgrades/overview) for details.
- **Notifications** — You can configure which users receive alerts for status changes on a specific environment. See [Notifications](/monitoring/notifications) for details.

<Note>
  Infrastructure-level settings — such as region, compute size, and network configuration — are managed by Redwood. Submit a support request if you need changes to those settings.
</Note>

## Who can manage environments

Access to environment management features in the portal is controlled by your organization's portal roles:

- **Organization Admin** — Can view and manage all environments, modify environment settings, and configure upgrade windows.
- **Environment Admin** — Can view and manage the specific environments they have been granted access to.
- **Viewer** — Can view environment details and status but cannot modify any settings.

<Info>
  If you need to manage environments but do not have the required role, ask your Organization Admin to update your permissions. See [Roles and permissions](/users/roles-and-permissions) for details on what each role can do.
</Info>

## Contact Redwood Support for environment issues

For issues beyond what you can resolve in the portal — such as unexpected status changes, performance problems, or requests to provision or decommission an environment — contact Redwood Support:

1. From the portal, select **Support** in the navigation or footer to open a support request.
2. Provide the environment name, the product, and a description of the issue.
3. For urgent production incidents, use your organization's designated priority support channel if one has been established with your Redwood account team.

Redwood Support can also assist with environment-level questions specific to RunMyJobs, Finance Automation, or JSCAPE that go beyond portal configuration.
